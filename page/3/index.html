Redis分布式锁实现

发表于 2018-07-24 更新于 2022-06-20

分布式锁一般有三种实现方式：1. 数据库乐观锁；2. 基于Redis的分布式锁；3. 基于ZooKeeper的分布式锁。

我们最先做电商+零售的库存业务时，选用的是数据库乐观锁，但是在并发较大时性能不太乐观。后来重构考虑使用Redis来实现分布式锁，采用了Spring的RedisLockRegistry。在压测时发现单一销售业务下性能比较稳定，但是在采购、批发等业务里，动则一个单据就要给2000-3000款商品加锁，高并发下（重点指混合场景下的并发）RedisLockRegistry在此场景的的缺点就暴露出来了：不可批量获取和释放锁，我们订单下的所有商品加锁，会消耗大量的网络资源且获取整单商品锁的时间和不可控制。

于是我们在参照RedisLockRegistry的技术上，写了RedisBatchLockRegistry。再压测时，我们原先场景并发提高10倍的基础上，性能依然满足要求。

之前都是在印象笔记上记录自己的笔记、心得和踩过的坑，一直想转移到博客上。但由于没有挑到喜欢的开源博客系统，自己又有严重的强迫症不愿意妥协，加之工作也挺忙（这个是借口）一直没有落实。

Hexo是一个简单、快速、强大的基于 Github Pages 的博客发布工具，支持Markdown格式，有众多优秀插件和主题。
其实很多我们经常光顾的博客都是基于Hexo搭建的，比如阿里间件团队博客等，只是自己从来没有关注过页面最下面那行小字“由 Hexo 强力驱动”或者“Powered by Hexo”。
